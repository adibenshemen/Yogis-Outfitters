Tal Barak


Store name: Yogis Outfitters

What are we selling: Yoga clothes and essentials

Additional pages we added to our online store:

1. About Us - telling about our brand.

2. Help - FAQ (frequently asked questions) about our store.

3. Contact Us - let users tell us what they think about our online store, contact with our team if they need and we can get back to them by email.

4. Products pages - we added a page for each product in our store, showing it’s name, description, price, available sizes and add to bag button, together with amount of pieces to choose.

What was hard to do?

In this course I’ve learned javascript for the first time, so the whole project was quite challenging for me. Google and me became best friends, so I think that the most challenging part in the project was understanding how everything works, for example how the client side and server side communicate with each other. Also understanding how to work with files, how to read, write and save data in json files. Making tests was also hard, because I had to figure out how to do tests that required me to be logged in. In addition, I’ve encountered some bugs that at first I wasn’t sure why it happens, until I realized the importance of Asynchronous functions, and now I can say that I deeply understand where to add async/await.

My partner was Adi Ben Shemen 204841621.

Adi and I worked together the whole time. At first Adi was leading the frontend side and I was leading the backend side (because I was really enthusiastic about working with files, and doing the admin part). Together we figured out how to create dynamic pages - which was at first challenging, but today, after learning so much - I can debug easily things that do not work, and understand how to fix it.

The different routes our app supports:

1. Reaching admin pages only if you’re logged in as admin.

2. Reaching pages that require you to be logged in (My Bag, Checkout) only when you’re logged in.

3. In My Bag page, manage to increase/decrease the amount of the wanted products.

4. Creating dynamic pages: products pages and single product pages, my bag page, searching results pages, etc.

5. Adding and removing products from admin page

6. Searching for users in admin page

7. Register and login only with right credentials (register only if the email doesn’t exist already)

8. Subscribe with email - save emails in json file

9. Send contact message - and save it in json file

10. Logout - different cases if you were logged in or not

11. Remove members, subscribers and contact us messages (it was useful for the tests - but in general it’s good to have the option to do that)
